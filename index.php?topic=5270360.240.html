Received Signals  page

👉 A new tab "Signals" (coming from Tradingview or your bot) has been added to the Trading menu.

 The page of received Signals shows a table on the left side:

1. Date and time of the Signal
2. The action / status of the signal:

- "Received" - signal received but not yet processed by the account
- "Processing" - the signal has been received by the account and is being processed
- "Missed" - the signal is missed, because the order hasn't been created due to some limitations. See error in the "Messages / Titles" column
- "Open LONG" - a signal has opened a LONG position
- Open SHORT" - signal opened SHORT position
- "DCA LONG" - signal averaged LONG position
- "DCA SHORT" - a signal averaged SHORT position
- Decrease. LONG" - signal decreased LONG position
- Decrease. SHORT" - a signal decreased SHORT position
- Close. LONG" - "Close LONG" signal closes the LONG position
- "Close. SHORT" - a signal has closed the SHORT position
- Reverse LONG - a signal closed the LONG and opened the SHORT position in 1 order
- "Reverse SHORT" - a signal closed SHORT and opened LONG in 1 order
- "FLAT" - a signal will close all positions opened in this currency pair
- "Update" - the signal will update the module, which manages the position

3. currency pair name
4. Message (detailed action, for example "Create order tp-12345") and webhook name. Search by webhook name.
5. Signal identifier

The signal details block is displayed on the right side of the table:

1. Hook information and the option to go to the position in which the order was created or to update the settings.
2. When the position is opened, the order created by this signal will be highlighted in green
3. Message code, which was received in the signal
4. The configuration, which was generated by taking into account the hook settings in the terminal to create an order or update a position module.

We recommend everyone who works with the Tradingview signals to restart the account and refresh the page in the browser.
